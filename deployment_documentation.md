# RAAH - Project Documentation

RAAH is a professional, empathy-driven legal guidance platform designed specifically for women in Pakistan. It combines advanced AI technology with a human-centric marketplace to help users understand their rights and connect with verified legal professionals.

---

## 🚀 Technology Stack

### Frontend
- **Framework**: [React 18](https://react.dev/) with [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS 4.0](https://tailwindcss.com/)
- **Animations**: [Framer Motion](https://www.framer.com/motion/)
- **Icons**: [Lucide React](https://lucide.dev/)
- **Components**: [Radix UI](https://www.radix-ui.com/) (Headless UI)
- **Charts**: [Recharts](https://recharts.org/) (Admin Analytics)
- **Routing**: [React Router 7](https://reactrouter.com/)

### Backend
- **Runtime**: [Node.js](https://nodejs.org/)
- **Framework**: [Express.js](https://expressjs.com/)
- **Authentication**: [JWT](https://jwt.io/) (JSON Web Tokens) & [Bcrypt.js](https://github.com/dcodeIO/bcrypt.js)
- **API Interaction**: [Fetch API](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API)

### Database & AI
- **Primary Database**: [PostgreSQL](https://www.postgresql.org/) (Hosted on [Neon](https://neon.tech/))
- **Vector Database**: [Pinecone](https://www.pinecone.io/) (For RAG-based AI context)
- **AI Models**: 
  - Primary: [Google Gemini 1.5 Pro/Flash](https://deepmind.google/technologies/gemini/)
  - Fallback: [xAI Grok](https://x.ai/)
- **Video Conferencing**: [Daily.co](https://www.daily.co/)

## 🗺️ User & Lawyer Journeys

### 👩‍💼 The User Journey (Client)
1.  **Onboarding**: Land on a professional, trust-building Landing Page.
2.  **Intake Interview**: A guided, empathetic interview process to collect incident details.
3.  **AI Guidance**: Receive a personalized "Case Brief" generated by AI, micro-dosing legal information in plain language.
4.  **Marketplace**: View recommended lawyers matched based on the user's specific case type and location.
5.  **Consultation**: Request a meeting. Once accepted, a secure video link is generated.
6.  **Safety**: Access **Emergency Helplines** via a dedicated pulsing red button on the dashboard.

### ⚖️ The Lawyer Journey (Professional)
1.  **Registration**: Sign up and provide credentials for verification.
2.  **Dashboard**: Manage incoming consultation requests.
3.  **Case Briefs**: Review AI-generated summaries of client cases *before* accepting a meeting.
4.  **Peer Collaboration**: Request consultations with senior consultants for complex cases.
5.  **Community**: Write and publish legal blogs (subject to Admin approval).
6.  **Earnings**: Track platform fees and session payments.

### 🛡️ The Admin Journey
1.  **Verification**: Approve or reject new lawyer applications.
2.  **Content Moderation**: Review blog posts before they go live.
3.  **AI Control**: Switch between Gemini and Grok models; update system prompts in real-time.
4.  **Analytics**: Monitor platform growth, active consultations, and revenue.

## 🔧 Environment Variables (.env)

You must set these variables for the project to function:

| Variable | Description |
|---|---|
| `DATABASE_URL` | Neon/PostgreSQL Connection String |
| `GEMINI_API_KEY` | Google AI Studio Key |
| `GROK_API_KEY` | xAI Console Key (Optional fallback) |
| `PINECONE_API_KEY` | Pinecone Console Key |
| `PINECONE_INDEX` | Your Pinecone index name (default: `raah-aap`) |
| `DAILY_API_KEY` | Daily.co API key for video sessions |
| `JWT_SECRET` | Secret string for user sessions |
| `ENCRYPTION_KEY` | 32-character key for encrypting sensitive DB data |

## 🚢 Deployment Guide

### Option 1: Railway (Recommended for Fullstack)
Railway is the best choice because it handles the Node.js server, the database, and environment variables in one place.

1.  **Connect Repo**: Link your GitHub repository to [Railway.app](https://railway.app/).
2.  **Configure Environment**: Add all variables from the table above in the "Variables" tab.
3.  **Build Command**: Railway will automatically detect `npm run build` and `npm start`.
4.  **Static Files**: The server is already configured to serve the `dist/` folder. Ensure the build completes successfully.

### Option 2: Vercel (Frontend Only)
If you deploy the frontend on Vercel, you will need a separate backend hosting service.

1.  **Import Project**: Connect GitHub to Vercel.
2.  **Framework Preset**: Select "Vite".
3.  **Root Directory**: Keep it as default.
4.  **Environment Variables**: Add `VITE_` prefixed variables if you need them client-side, but most of RAAH's logic is server-side.
5.  **Note**: You must deploy the `server/` directory elsewhere (like Render or Railway) and update the frontend fetch URLs.

## 📋 Pre-Deployment Checklist
- [ ] Run `npm run build` locally to ensure no TypeScript/Vite errors.
- [ ] Verify `DATABASE_URL` has `ssl=true` or `rejectUnauthorized: false` for Neon.
- [ ] Ensure the first user registered becomes an Admin (or manually update the `role` in the DB).
- [ ] Check that all background images in `public/` are correctly referenced.
